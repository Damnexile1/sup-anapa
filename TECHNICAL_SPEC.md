# Техническое задание: SUP-Anapa

## Описание проекта
Веб-сервис для бронирования SUP-прогулок (Stand Up Paddleboarding) с различными инструкторами в Анапе.

## Функциональные требования

### 1. Публичная часть (для клиентов)

#### 1.1 Главная страница
- Галерея фотографий с SUP-прогулок
- Кнопка "Забронировать прогулку"
- Информация о сервисе

#### 1.2 Страница бронирования
- Отображение доступных слотов (дата + время)
- Фильтрация по датам
- При выборе слота показывается:
  - Погодные условия (ветер, солнечность, температура воздуха и воды)
  - Доступные инструкторы
  - Цена прогулки
- Форма бронирования с контактными данными клиента

#### 1.3 Подтверждение бронирования
- После создания брони отправляется уведомление администратору
- Статус брони: "Ожидает подтверждения"
- Финальное подтверждение через администратора (future feature)

### 2. Административная панель

#### 2.1 Аутентификация
- Вход по логину/паролю
- Сессии для администраторов

#### 2.2 Управление инструкторами
- Добавление/редактирование/удаление инструкторов
- Поля: имя, фото, описание, контакты

#### 2.3 Управление слотами
- Создание временных слотов для бронирования
- Настройка: дата, время начала, время окончания, цена
- Привязка инструкторов к слотам
- Максимальное количество участников на слот

#### 2.4 Управление бронированиями
- Просмотр всех броней
- Подтверждение/отклонение броней
- Статусы: ожидает подтверждения, подтверждено, отменено

### 3. Интеграции

#### 3.1 Погодный API (OpenWeatherMap)
- Получение прогноза погоды на выбранную дату
- Данные: температура воздуха, температура воды, скорость ветра, облачность
- Кэширование данных для оптимизации запросов

#### 3.2 Уведомления (VK Bot API)
- Отправка уведомлений администратору при новом бронировании
- Архитектура должна поддерживать легкое добавление Telegram в будущем
- На первом этапе - заглушка с логированием

## Технический стек

### Backend
- **Язык**: Go 1.21+
- **Фреймворк**: Chi router (легковесный, идиоматичный)
- **БД**: PostgreSQL 15+
- **Миграции**: golang-migrate
- **ORM/Query builder**: sqlx (легковесный) или GORM (если нужен полноценный ORM)

### Frontend
- **Шаблонизатор**: Go html/template
- **Интерактивность**: htmx для динамических обновлений
- **Стили**: Tailwind CSS (через CDN на первом этапе)
- **JS**: Минимальный vanilla JS где необходимо

### База данных

#### Основные таблицы:
1. **instructors** - инструкторы
2. **slots** - временные слоты для бронирования
3. **bookings** - бронирования
4. **admins** - администраторы системы
5. **weather_cache** - кэш погодных данных

### Инфраструктура
- **Контейнеризация**: Docker + Docker Compose
- **Деплой**: VPS (подготовить docker-compose.yml)
- **Веб-сервер**: Nginx как reverse proxy
- **SSL**: Let's Encrypt (настройка в будущем)

## Этапы разработки

### Этап 1: Инфраструктура (MVP)
- Инициализация Go проекта
- Настройка PostgreSQL схемы
- Базовая структура приложения
- Docker setup

### Этап 2: Админка
- Аутентификация
- CRUD для инструкторов
- CRUD для слотов
- Просмотр броней

### Этап 3: Публичная часть
- Главная страница
- Страница выбора слотов
- Интеграция с погодным API
- Форма бронирования

### Этап 4: Уведомления
- Интеграция с VK Bot API
- Отправка уведомлений при бронировании

### Этап 5: Полировка
- Улучшение UI/UX
- Оптимизация производительности
- Тестирование
- Документация

## Безопасность
- Хеширование паролей (bcrypt)
- CSRF защита для форм
- Rate limiting для API
- Валидация всех входных данных
- Prepared statements для защиты от SQL injection

## Мониторинг и логирование
- Структурированное логирование (zerolog или slog)
- Логирование всех критических операций
- Метрики (опционально: Prometheus)
